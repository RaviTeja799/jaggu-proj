## How the Knowledge Base is Used

### 1. Semantic Matching
Each requirement's description and keywords are converted to embeddings using Sentence Transformers. When a clause is analyzed:
- The clause text is converted to an embedding
- Cosine similarity is calculated against all requirements
- Matches above threshold (0.75) are considered relevant

### 2. Keyword Matching
The system also uses rule-based keyword matching:
- Each requirement has 5-15 relevant keywords
- Clauses are scanned for keyword presence
- Multiple keyword matches increase confidence

### 3. Mandatory Element Checking
For matched requirements, the system checks if all mandatory elements are present:
- **Complete:** All elements present
- **Partial:** Some elements present
- **Non-Compliant:** Missing critical elements

### 4. Risk Scoring
Based on:
- Requirement risk level (High/Medium/Low)
- Compliance status (Compliant/Partial/Non-Compliant)
- Number of issues found
- Confidence score
